# Gastropos SDD

Este repositorio utiliza Spec Driven Development.

## Estructura

- specs/: contratos del sistema
- agents/: definición de agentes
- skills/: capacidades reutilizables
- milestones/: roadmap ejecutable
- execution/: trazabilidad del estado y decisiones del SDD

## Regla

Nada se implementa sin estar definido en specs/milestones.

## Seguimiento de ejecución

Los archivos dentro de `execution/` deben actualizarse siguiendo `specs/operations/execution-tracking.md` cada vez que se inicia o cierra un hito.
