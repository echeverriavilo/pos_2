# Validation Rules

## Principle

Una implementación es válida solo si cumple:

- tests
- specs
- calidad de código

---

## Failure Conditions

Una implementación es inválida si:

- faltan tests
- tests están incompletos
- hay inconsistencia con specs
- el código es poco limpio
- naming incorrecto

---

## Validation Priority

1. Tests
2. Specs
3. Código

---

## Testing Requirement

- todos los tests deben pasar
- deben cubrir reglas de negocio

---

## Spec Alignment

- toda lógica debe mapear a specs
- no se permite comportamiento implícito

---

## Code Quality

Debe cumplir:

- coding-standards.md
- claridad
- consistencia

---

## Stop Condition

Si cualquier validación falla:

→ no se puede avanzar

---

## Success Criteria

- implementación validada en todos los niveles
